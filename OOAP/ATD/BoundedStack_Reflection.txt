BoundedStack
1) статусы команд - в своей реализации неверно указал описание для операций, а именно

было 
POP_NIL // pop() ещё не вызывалась 
PEEK_NIL // peek() ещё не вызывалась

должно быть
POP_NIL // push() ещё не вызывалась 
PEEK_NIL // push() ещё не вызывалась

Поскольку предусловия для работы данных операций - "стэк не пустой", то именно команда push определяет выход из этого статуса для указанных операций.

2) метод clear() - в моем АТД отсутствует постусловие - "из стека удалятся все значения". Поскольку команда меняет состояние объекта наличие постусловия является обязательным для полноты спецификации АТД

3) синтакситечки отличается предусловие метода push:
в моем АТД:
в стеке есть свободное место (размер меньше максимального)
в примере:
в стеке менее максимального кол-ва элементов

Однако семантически предусловия идентичны, поэтому корректировки не требуется
