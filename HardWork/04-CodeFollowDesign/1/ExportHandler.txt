Дизайн класса ExportHandler
Цель - начать экспорт данных

Единственный метод Handle - начинает экспорт

Предусловия
    - нет
    
Постусловия
    - если задание есть в БД -- начинает экспорт
    - если задания нет -- ничего не выполняет
    

Рефлексия
В целом исходный класс (метод) выполнял свою задачу и возможно пример несколько надуманный, 
однако описание дизайна показало, что класс должен лишь запускать экспорт данных,
а в методе handle происходило получение и обработка задания запуска

Поэтому согласно требованию соответствия дизайна 1:1, я оставил в исходном классе лишь запуск процесса
экспорта -- для чего ввёл новую абстракцию - exportProcessor.
Сам процессор взял на себя логику получения и обработки задачи экспорта
Однако при этом была убрана проверка на null если задача не найдена, вместо этого в фабрику, возвращающую стратегию
экспорта добавил NullStrategy -- которая определяет, что делать с пустыми задачами

Также использовал библиотеку microsoft Contracts -- аналог Java библиотеки JML, который нашёл для C#
С помощью неё выразил рпед и постусловия нового метода
