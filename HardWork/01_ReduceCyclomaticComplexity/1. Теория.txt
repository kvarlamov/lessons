Жесткие Механизмы снижения ЦС:
1) Никаких условных операторов и switch-case
2) Никаких циклов (for, while)
3) Никаких Null/None

Чем выше ЦС - тем более закрытый код


Мягкие правила снижения:
1) запрещены else, else if
2) запрещены циклы, вложенные в if, if inside if
3) Внутри цикла только 1 if -- и то для прерывания работы (continue, break, return)
4) в каждом if обрабатываем лишь 1 аргумент, а не несколько

Самое качественное решение - ad-hoc полиморфизм
Единый интерфейс с разной реализацией - полиморфизм подтипов

Если в некотором методе есть условие, выполняющее качественно разный код -- создать разные методы с одним названием


Multiple dispatch - методы имеют одинаковое кол-во параметров, но разный тип (в рамках полиморфизма например)


Паттерн табличной логики  -- инжектим например Func<TaskType, IProcessor>

Можно также использовать перегрузку методов, рефлексию, instanceof, паттерн Visitor и т.п.
