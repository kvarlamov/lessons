Пример 2.
Исходная ЦС  = 11
Финальная ЦС = 3

Проблема:
В исходном методе наблюдалось большое количество цепочек if .. else if ...else,
при этом if были вложены в другие if, что и давало такую ЦС, а также ухудшало
читаемость и возможность расширения кода

Для рефакторинга применил следующие методы и приемы:
Для рефакторинга была в сущности заказа был выделен тип заказа, с помощью
которого через класс-фабрику можно получить нужную стратегию для обработки (паттерн стратегия).

Также был выделен общий АТД заказа, имеющий единственный метод обработки,
при это внутри него вызывался абстрактный метод Process, которые переопределялся
в каждом потомке (полиморфизм), а также выбиралась нужная стратегия. Метод GeneralOrderProcess
представляет реализацию паттерна "Шаблонный метод".

В конечном итоге метод обработки заказов пользователя сократился до одного 
цикла, в котором каждый заказ, в зависимости от его реального типа, обрабатывается по 
разному.
